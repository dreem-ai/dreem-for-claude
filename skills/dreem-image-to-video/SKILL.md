# Dreem Image-to-Video

A still becomes a 5-10 second clip.

## Before you start

- `../../shared/references/_shared-input-analysis.md` - Step 0 and the hard rules. HARD RULE 2 is this skill's gate. Read FIRST.
- `../../shared/references/_shared-dreem-mcp-guide.md` - flows, sourcing, async pattern.
- `../../shared/references/_shared-channel-specs.md`, `../../shared/references/_shared-credit-model.md`, `../../shared/references/_shared-output-conventions.md`.

## Step 1 - The source still (the gate)

LOOK at the source still before anything else (Step 0 in `_shared-input-analysis.md`). **The first frame must be an on-model image** (HARD RULE 2). A flat lay or ghost mannequin in motion reads as a floating AI garment - do not generate from it. If the input is not on-model, stop and offer the real path: generate the on-model shot first via `dreem-virtual-model-shots` (or both steps in one `dreem-content-kit` run), then animate that output.

**Baby/infant garments: no video at all** (HARD RULE 3) - there is no legitimate on-model first frame to build from. Route to `dreem-product-shots`.

Chaining is the intended pattern: an output URL from `dreem-virtual-model-shots` is a public HTTPS URL and works directly as the video source. Match the still's shape to the destination - a `Ratio9x16` still for Reels/TikTok/Stories, `Ratio3x4` or `Ratio16x9` for PDP and site - and default to the ratio the product's gallery already uses when no destination is named (FORMAT RULE).

**On-model but wrong ratio** (e.g. a 1:1 still bound for 9:16 Reels): offer both paths with exact costs - animate as-is at the source ratio, or generate a virtual-model reframe at the target ratio first and animate that output. Never crop, never quietly ship the wrong shape into a vertical placement.

Send as `images[]` with `frameType: FirstFrame`. Optionally add a `LastFrame` image to pin the end state - a front-to-back move requires a real back view as the `LastFrame`.

## Step 2 - The motion

Exactly one of:

- `promptId` - a curated motion style from `browse_video_styles` (semantic query: "slow turn", "gentle push-in", "wind in fabric"). The picker shows looping previews; best default for non-directors.
- `prompt` - freeform creative direction when the user describes specific motion, camera, lighting, or mood.

When using `promptId`, follow **Casting, pose, and motion: suggest, explain, confirm** in the MCP guide: pick the best-fit style, explain it in one line tied to the destination, ask the user to confirm or pick another. If the picker's live state later shows an unclicked default that isn't what you recommended, that is the picker's own pre-highlight, not the user's answer - stick with the confirmed pick unless the user actually clicked something else or named a different one in chat.

Guidance by destination: PDP/site wants subtle (soft push-in, gentle weight shift - premium reads quiet). Social wants visible movement (fabric motion, energy). Duration: `5s` default; `10s` when the destination is social storytelling.

**No destination named? Assume PDP.** Subtle motion, `5s`, the gallery's existing ratio - and state the assumption in the plan ("planned for your product page - say the word if this is for TikTok/Reels and I'll switch to bolder motion at 9:16") so one word flips it. Don't ask a separate destination question.

Turn/360/rotation motion is a back view in disguise (HARD RULE 1): pick it only when a back-view input exists, or after the user explicitly acknowledges the back will be made up. Otherwise stay front-facing.

## Step 3 - Cost preview, then the gate

Video costs more credits than stills - say so in the plan table (`_shared-credit-model.md`). Interactive: pickers first, `confirm` LAST (RULE-A), stop, wait. Batch: plan table + explicit "go".

## Step 4 - Generate and wait properly

`generate_product_video` with `images`, `duration`, and `prompt` OR `promptId`. Expect \~120s for a 10s clip - keep calling `wait_for_content` with the same requestId until `status='completed'` AND a non-empty output URL exists (a `completed` with empty outputs is not done). Then `show_generation_result` exactly once.

## Step 5 - QA and hand over

Watch the clip: garment warping, melting logos, extra fingers, physics glitches in fabric. Flag honestly in `manifest.csv` (`qa_flag`), suggest a re-run with a calmer motion style for bad rows (subtle prompts fail less). Write `summary.md` with per-destination delivery notes.
